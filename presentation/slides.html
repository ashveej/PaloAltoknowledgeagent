Introduction

My Technical-PM operating system

1

Slice the risk

Ship the thinnest vertical that retires the riskiest assumption first. Here it's trust, not "can an LLM answer" — so the first slice was the whole trust loop.

2

Determinism where trust lives

The LLM classifies, drafts, and styles. It never decides confidence, citations, or whether to answer. Those are auditable Python.

3

Make trust legible

"Medium" is useless. "Medium because the only source is 241 days old" changes how an SE uses it. Every score is explained; every claim shows its source.

Context & Problem

The user, and the asymmetric cost of being wrong

Technical Sales / SEs, live in a customer call:

"Does Prisma Access support X in version Y?"

  • Confident-but-wrong loses credibility and the deal
  • Truth is sprawled across guides, release notes, KB, Confluence — and goes stale
  • SMEs answer the same Qs in Slack; that knowledge never becomes reusable

Why a generic chatbot makes it worse

It hallucinates with confidence and leaves no audit trail. For a security vendor, an unsourced claim is a liability — not a feature.

The task is an evidence-governed RAG system with a human-in-the-loop knowledge flywheel.

Product principles

Four non-negotiables

  • No citation → no answer. Every factual claim maps to approved source text, or it doesn't ship.
  • Confidence is calculated, not generated. The LLM never sets it.
  • Brand voice is a presentation layer. It can clarify; it can't add claims or drop caveats.
  • SME answers become governed knowledge. Not a Slack reply — an owned, citable, review-dated asset.

Technical Deep Dive · the key idea

The Facts ↔ Style wall

Facts engine

Touches evidence. Produces validated claims + caveats + a calculated confidence. Never styles.

VALIDATED CLAIMS ONLY →

Style layer

Never sees raw docs. Rewrites in brand voice, then is re-checked: no new claims, no dropped caveats, citations intact, banned phrases blocked.

This is how I answer the classic ask — "make it less robotic" — without risking accuracy. Voice and trust are scored separately; voice can never raise trust.

Data Strategy · acquisition

Three sources, deliberately

Real docs

A sub-agent crawled docs.paloaltonetworks.com (Prisma Access, Cortex XDR, NGFW) → Markdown+frontmatter. ~17 pages, 68 chunks.

PDFs

pypdf ingestion + metadata sidecar. SEs live in datasheet PDFs.

Synthetic

Hand-seeded edge cases I needed to exist: a stale doc, a deprecated doc. You can't test a freshness cap without a stale document.

Hard truth  The Cortex portal is JS-rendered — the crawler got nothing. I named the coverage gap instead of faking it. That gap is why a Cortex exploit-protection question correctly escalates in the demo.

Data Strategy · quality & alignment

Metadata is a first-class citizen

Every chunk carries product, feature, version, source_type, owner, last_reviewed_date, deprecated, approval status, and votes.

It drives retrieval

Deprecated / unapproved / off-product chunks are never surfaced — a gate at the data layer.

It drives trust

Source authority + freshness feed the score; downvotes feed both ranking and the confidence cap. Metadata isn't decoration — it's the governance.

Human in the Loop

The interface between AI logic & the SE

  • Attribution UI — the answer embeds the exact cited chunk text inline. Verify the AI's work without leaving the page.
  • Explainable confidence — pill + score breakdown + the cap reason, all visible.
  • Feedback → ranking, not retraining — "wrong answer" lowers source reputation (feeds rerank + cap) and opens an SME ticket.
  • SME flywheel — escalate → feature-level routing → email the SME group → Approve & Publish → instantly citable.
  • Brand voice — a separate, brand_voice.yaml-configurable deterministic layer (terminology, guards, scored rubric).

Evaluation & Results · the hard truths

What broke — and what it taught me

  • Cross-encoder logits aren't calibrated. My absolute relevance floor was wrong — the correct chunk scored −1.0 while an off-topic one scored +1.5. Rerankers are for ordering; the off-topic gate belongs to the LLM.
  • Opus 4.8 deprecates temperature. Chasing determinism, I set it → 400 → my fail-closed design silently escalated all 15. Determinism belongs in the Python layer, and swallowed errors need alerts.
  • A fail-safe can become bad UX. The facts engine over-escalated answerable questions. Since the gate already stops hallucination, I biased toward answering. Trust = answering when grounded.
  • Determinism has sharp edges. A normalizer produced "Palo Alto Networks Networks." Small bug — exactly why the deterministic layer is unit-tested.
Retrospective

If I rebuilt it on today's stack

Would change

  • Agentic retrieval — query decomposition + Corrective/Self-RAG: critique evidence, re-query before answering
  • Tool-using agent for live doc lookup before escalating
  • LLM-as-judge as a 2nd eval signal (never for user-facing confidence)
  • Voyage embeddings + hybrid + fine-tuned reranker on feedback
  • Real connectors (Confluence/Jira) + streaming UX + agentic SME drafting

Would NOT change — the spine

  • Deterministic confidence
  • Hard citation gates
  • The Facts ↔ Style wall

These earned the trust. Everything else is an optimization.
